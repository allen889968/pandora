button 按钮兼容性研究

按钮在网页中必不可少并广泛应用，且在表单布局中面临较为复杂的兼容性问题，此处对按钮实现统一兼容解决方案。

规划：

  • 首先 (5种)不同标签 实现兼容性按钮，（a、span button 前期实现）（input[type="button"]、input[type="submit"] 后期制作）
  • 之后制定 尺寸4种、样式N种、交互状态4种
  • 新增按钮 loading 效果，交互变色应用等
  • 扩展主题风格：经典的渐变实现(默认)、扁平化的metro风格
  • 问题总结，buttons 使用中可能遇到的问题及解决办法

注意：此处只针对以上规划的全兼容性做一研究，全部实现请到模块 buttons.html 中浏览效果。

可能遇到的问题：

  • A标签 提交表单IE6下事件冲突
  • 为什么不用图片按钮？图片按钮虽能全兼容，但适应性扩展性都不高
  • 此处实现的按钮哪里会用到？用于较独立的位置，区别于弹窗上附带的按钮，但仍让可以组合使用
    理论上引用组件css的顺序是：小组件先引用如button.css，大组件后引用如dialog.css
  • 实验测试dev版本 button-dev.html

实现规划：为测试完整的兼容性问题，以下分步实现，首先制作 a 标签按钮，之后为其他标签，依次类推。

以上实现的事项说明

  1. 推荐使用 a 标签实现按钮
  2. 针对IE6/7，关闭了渐变的实现（避免影响性能）采用平面实现，a标签在IE6下若无href属性，hover效果失效
  3. 针对IE6，组合icon时导致btn的line-height失效，增加了btn-text标签放置文字
  4. 为增强交互效果，对边框做了视觉处理，但IE6/7的button等标签实现由于聚焦时出现不可控的黑边，设置为无边框
  5. 实现的按钮默认设置右边距6px，结合换行或空格产生的间距，总体效果表现为10px，具体应用环境可自行调整

问题描述

火狐下 button 及 input 标签无视 line-height，且button和input[type="button"],input[type="submit"],input[type="reset"]等input按钮，它就是border-box模型。

这里还存在这么一个问题，即便是在火狐下把按钮（非链接）声明为content-box;但是按钮依然存在左右3个像素的隐形填充，无论字体设置多大。一个解决办法是使用-moz-padding-start和-moz-padding-end来改变。

问题

  • input实现的按钮，IE6/7下宽度变宽许多，原因待研究，IE9-11变得稍微窄了点(<5px数量级)，可能由字体宽度引起
  • 高度上已经表现的想当一致，IE6/7表现的水平方向位置差了1px，有待做垂直居中时进一步研究

调整

  • 按钮默认尺寸

    为了实现更多场景的变化，默认的尺寸为紧凑型，按钮内边距较小，且内边距随按钮变大而增大

    新调整尺寸大小：
    默认按钮：padding: 4px 10px;(h30px) font-size: 12px; font-weight: normal;
迷你按钮：padding:   0  6px;(h22px)
　小按钮：padding: 2px 10px;(h26px)
　中按钮：padding: 4px 12px;(h30px)
　大按钮：padding: 7px 15px;(h36px) font-weight: bold;
超大按钮：padding: 2px 20px;(h42px) font-weight: bold; 暂时不用此尺寸

    按钮尺寸调整如上标红部分，可以细微变更，但变更较大，应根据需求定制需要的主题样式，如：.lvstyle 进行整体调整， .btn-w 调整左右内边距为20px

  • 按钮类型

    灰色按钮(深浅两种色)都表示禁用按钮，其他按钮有半禁用状态(正在响应中不要继续点击)
