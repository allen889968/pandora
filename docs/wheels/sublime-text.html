Sublime Text 安装使用

首先下载 Sublime Text 2、Sublime Text 3，可以根据你的系统选择，Sublime Text支持跨平台，Window平台和Mac甚至Linux，你无需为以后变更平台后要改变编辑器操作习惯而头疼。

使用 Package Control

Package Control即安装包控制器，有了这个就可以使用丰富的插件包了。安装方法有两种，推荐第一种：

1、访问Package Control站点按照提示复制一段python命令安装Package Control，按Ctrl + ` 调出console控制台，粘贴对应的命令行回车即可。

SUBLIME TEXT 3

import urllib.request,os; pf = 'Package Control.sublime-package'; ipp = sublime.installed_packages_path(); urllib.request.install_opener( urllib.request.build_opener( urllib.request.ProxyHandler()) ); open(os.path.join(ipp, pf), 'wb').write(urllib.request.urlopen( 'http://sublime.wbond.net/' + pf.replace(' ','%20')).read())

SUBLIME TEXT 2

import urllib2,os; pf='Package Control.sublime-package'; ipp = sublime.installed_packages_path(); os.makedirs( ipp ) if not os.path.exists(ipp) else None; urllib2.install_opener( urllib2.build_opener( urllib2.ProxyHandler( ))); open( os.path.join( ipp, pf), 'wb' ).write( urllib2.urlopen( 'http://sublime.wbond.net/' +pf.replace( ' ','%20' )).read()); print( 'Please restart Sublime Text to finish installation')

重启Sublime Text，如果在 Perferences->package settings 中看到 package control 这一项，则安装成功。

如果这种方法不能安装成功，可以使用第二种方法下载编译包手动安装。

下载 Package Control 插件包到插件目录

插件目录在菜单中打开 Preference--Browse Packages，没有自己新建一个

使用Package Control安装插件的方法：

1、按下Ctrl+Shift+P(Win) 或 Command+Shift+P(Mac) 调出控制面板

2、输入install调出Install Package选项并回车，然后在列表中选中要安装的插件。

常用的命令Install Package (安装扩展)、List Packages (列出全部扩展)、Remove Package (移除扩展)、Upgrade Package (升级扩展)

方法介绍完毕，下面推荐一些常用插件：

Sublime Text 精华插件

  • Emmet

    就是 Sublime Text 版本的 Zen Coding，改名 Emmet 了。前端开发必备插件，Write less , show more，Tab键触发。还不知道ZenCoding，你要补课了《 Zen Coding: 一种快速编写HTML/CSS代码的方法 》

  • Tag

    Html格式化，右键Auto-Format Tags on Ducument。

  • GBK to UTF8

    将文件编码从GBK转黄成UTF8，快捷键Ctrl+Shift+C

  • Sublime Prefixr

    Prefixr，CSS3 私有前缀自动补全插件，Ctrl+Alt+X触发。

  • JS Format

    一个JS代码格式化插件。

  • SublimeLinter

    支持lint语法的插件，可以高亮linter认为有错误的代码行，也支持高亮一些特别的注释，比如“TODO”，这样就可以被快速定位。

  • Alignment

    代码对齐，如写几个变量，选中这几行，Ctrl+Alt+A，就对齐了。注意会和QQ的截图快捷键冲突。

  • Bracket Highlighter

    类似于代码匹配，可以匹配括号，引号等符号内的范围。

  • Hex to HSL

    自动转换颜色值，从16进制到HSL格式，快捷键 Ctrl+Shift+U

  • Git

    基本上实现了git的所有功能。

  • SFTP

    直接编辑 FTP 或 SFTP 服务器上的文件

  • Clipboard History

    剪贴板历史记录，显示更多历史复制，Ctrl+Shift+V触发。

  • HtmlTidy

    清理与排版你的HTML代码

  • YUI Compressor

    压缩JS和CSS文件

  • Theme *

    Soda、Flatland等都是最常用的主题

  • SideBarEnhancements

    侧栏右键功能增强，非常实用

  • DeleteBlankLines

    删除空行

如果打开中文会有乱码，原因是ST2本身只支持UTF-8编码，而遇到的中文文件是GBK或者GB2312等编码时就会出现乱码，解决方法是安装一个 ConvertToUTF8 或者 GBK Encoding Support 插件即可完美解决，这个不是什么大问题。

常用配置

在 在菜单Preferences -> Settings- User 中设置修改。

{
    "line_numbers": true,      //是否显示行号
    "gutter": true,             //是否显示边列
    "auto_indent": true,        //是否自动缩进
    //"wrap_width": 170,        //换行宽度(单位：字符)
    "scroll_past_end": true,    //滚动能否超过结尾
    "font_face": "courier new", //等宽字体
    "tab_size": 4,              //TAB缩进宽度
    "translate_tabs_to_spaces": true,   //自动转换TAB为空格
    "update_check": false,      //是否检测版本升级
    "use_tab_stops": true,      
    "auto_match_enabled": true,
    "draw_white_space": "all",
    "font_size": 19,
    "gutter": true,
    "highlight_line": true,
    "word_wrap": true          //是否自动换行
}

注意

带有后缀Default的，为默认设置，每次升级插件都会重置这里的设置。所以尽量不要修改这里，否则升级会丢失你原先的设置。

带有后缀User的，为用户自定义设置，你可以把Default里面的设置全部复制一份到这里，然后再修改，这里存在的设置选项会覆盖Default里面的，即User的优先级更高。

sublime text 快捷键列表

有了上面的介绍，也入门了，那么剩下的就是熟练快捷键了，有了这些快捷键，你就有了神速高效率。这里我们尽量简单些介绍，方便记忆。下面这些是我最喜欢的高效快捷键。

  • Ctrl+G：跳转到第几行
  • Ctrl+R：在html文件里是 id 关键字导航；css中是每条规则导航；js里这是每个function导航
  • Ctrl+Shift+P：打开命令面板
  • Ctrl+P：搜索项目中的文件，这是sublime上面很好用的功能之一。直接输入输入关键字，可以在已打开的项目文件夹中进行快速文件名导航。
    并且输入:前缀即为(Ctrl+G),加上@就是(Ctrl+R)了
  • Ctrl+`：可以调出命令行模式(view->show console)，主要支持python语法等
  • Ctrl+Z：撤销 Ctrl+Y 恢复撤销
  • Ctrl+D：选择单词，重复按可增加选择下一个相同的单词
  • Ctrl+L：选择整行，重复可依次增加选择下一行
  • Ctrl+X：删除当前行
  • Ctrl+Shift+L：选择多行
  • Ctrl+Shift+V：粘贴并格式化
  • Ctrl+F：是查找，点击 Find All 后，则一下选择本页面的全部相同的文本，然后按键盘替换则全替换
  • Ctrl+H：替换
  • Ctrl+Shift+F：查找并替换 默认是在本项目(用得最多的)．也可能来文件夹里查找，与普通编辑不同的地方是sublime允许添加多个文件夹进行查找(add directory)
  • Ctrl+L：选择整行（按住-继续选择下行，即按住Ctrl不放按一次L则多选一行）
  • Ctrl+Shift+K 删除整行,Ctrl+KK 从光标处删除至行尾
  • Ctrl+/：注释当前行
  • Ctrl+Shift+/：块注释(注释已选择内容)，sublime会自动根据语言选择注释方式
  • Ctrl+Enter 当前行后插入行 Ctrl+Shift+Enter 当前行前插入新行
  • 在页面右下角，有快速设置tab转为spaces的菜单，可以指定页面的tab宽度，也可以将页面所有tab转为指定的spaces格数
  • Shift+右键拖动：选择多行，用来更改或插入列内容
  • 按Ctrl，依次点击或选取，可需要编辑的多个位置
  • Ctrl+Shift+↑或↓ 上下行互换
  • Ctrl+K Backspace 从光标处删除至行首
  • Ctrl+Shift+D 复制当前行，插入在该行之前

其他一些快捷键

以下这些我不太常用，也许你会用得到。

  • Ctrl+F2：设置/删除标记，然后F2在标记之间切换，Shift+F2 上一个标记。这里不爽的是无法进行鼠标添加标记
  • Ctrl+W：关闭当前打开文件
  • Ctrl+Shift+W：关闭所有打开文件
  • Ctrl+M：跳转到对应括号
  • Ctrl+U：软撤销，撤销光标位置
  • Ctrl+J：选择标签内容
  • Ctrl+H：替换
  • Ctrl+N：新建窗口
  • Ctrl+K+B：开关侧栏
  • Ctrl+Shift+M：选中当前括号内容，重复可选着括号本身
  • Ctrl+Alt+/：块注释，并Focus到首行，写注释说明用的
  • Ctrl+Shift+A：选择当前标签前后，修改标签用的
  • F11：全屏
  • Shift+F11：全屏免打扰模式，只编辑当前文件
  • Alt+F3：选择所有相同的词
  • Alt+.：闭合标签
  • Alt+Shift+数字：分屏显示
  • Alt+数字：切换打开第N个文件
  • 鼠标的前进后退键可切换Tab文件
  • ctrl+up, ctrl+down可以一行一行的滚动屏幕
  • pageUp pageDown一页一页翻动
  • ctrl+shift+a 可以HTML标签内容，每按一次逐级往上
  • ctrl+shift+space 可以选择js等语言里的块{内容}
  • Ctrl+J 合并行（已选择需要合并的多行时）
  • Ctrl+KU 改为大写
  • Ctrl+KL 改为小写
  • Ctrl+M 光标跳至对应的括号(BracketHighlighter插件做得更好)
  • Ctrl+Shift+M 选择括号内的内容（按住-继续选择父括号）
  • Alt+. 闭合当前标签
  • Ctrl+Shift+A 选择光标位置父标签对儿
  • Ctrl+Shift+[ 折叠代码
  • Ctrl+Shift+] 展开代码
  • Ctrl+KT 折叠属性
  • Ctrl+K0 展开所有
  • Ctrl+U 软撤销
  • Ctrl+T 词互换
  • Tab 缩进 自动完成
  • Shift+Tab 去除缩进
  • 在 菜单View-syntax下可以将当前面面以指定的语言进行语法高亮，通常情况下sublime会自动进行判断选择
  • snippets 代码片段插件，在tools-snippets菜单下可以查看当前页面支持的snippets，可以在缩略词后加tab自动补全snippets
  • Build编译，可以进行ruby等代码的编译，我安装了一个yui compressor后，就可以在css js页面里进行压缩，
    快捷键是ctrl+B，会自动在当前目录下生成xx.min.js、xx.min.css等压缩文件，这个很实用
  • 添加智能代码块提示 Tools->new snippet (mac机下保存后缀要为.sublime-snippet ) content：放代码必须放在CDATA里，$1代表光标位置，方便编辑类名 tabTrigger：快捷键 scope: 响应范围 可以是 html css js文件等 description：代码描述 在输入快捷键的时候 提示框内的介绍描述文字
